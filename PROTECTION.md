Since these settings cannot be configured via code, please follow these steps manually on GitHub.com.

## 1. Branch Protection Rules
1.  Go to your repository on GitHub.
2.  Click **Settings** (Top Tab) ‚Üí **Branches** (Left Menu) ‚Üí **Add branch protection rule**.
3.  **Branch name pattern**: `main`
4.  Check the following boxes:
    *   [x] **Require a pull request before merging**
        *   [x] *Require approvals* (Set to 1 or more)
        *   [x] *Require review from Code Owners* (Enforces CODEOWNERS file)
    *   [x] **Require status checks to pass before merging**
        *   (Search for `build`, `test`, or your specific CI job names)
    *   [x] **Do not allow bypassing the above settings** (Optional: Prevents admins from cheating)

## 2. General Settings
1.  Go to **Settings** ‚Üí **General**.
2.  Under **Pull Requests**:
    *   [x] *Allow squash merging* (Recommended for clean history)
    *   [ ] *Allow merge commits* (Disable if you prefer linear history)
    *   [ ] *Allow rebase merging* (Advanced users only)

## 3. Secret Scanning (Security)
1.  Go to **Settings** ‚Üí **Code security and analysis**.
2.  Enable **Secret scanning** (Alerts you if you accidentally commit API keys).
3.  Enable **Dependabot alerts** (Warns you about vulnerable dependencies in `package.json`).
